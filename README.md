# Синтетический motion-концепт

Проверяемый HTML/GSAP-пример для отклика на задачу с четырьмя анимированными баннерами. Это не клиентский кейс: визуал и текст придуманы специально для демонстрации процесса.

Показано:

- асимметричная композиция 1440×560;
- один фирменный визуальный язык для серии;
- детерминированная временная шкала 8,5 секунды;
- последовательные входы смысловых блоков, стабильная hero-фаза и мягкий финал;
- отсутствие внешних фотографий и спорных лицензий.

## Результат

- [`motion-banner-demo.mp4`](motion-banner-demo.mp4) — финальный ролик, 1440×560, 30 fps, H.264;
- [`index.html`](index.html) — исходная HTML/GSAP-композиция;
- [`hero-frame.png`](hero-frame.png) — контрольный статичный кадр.

Проверено HyperFrames: lint/runtime/layout/motion — без ошибок и предупреждений, 96 из 96 проверок контраста проходят WCAG AA. Финальный MP4 содержит 255 кадров; дополнительно проверен автоматическим детектором крупных чёрных областей.

Для воспроизводимого рендера на Windows здесь отключены две экспериментальные оптимизации захвата кадров:

```powershell
$env:HF_STATIC_DEDUP='false'
$env:PRODUCER_ENABLE_STREAMING_ENCODE='false'
npx hyperframes render --output motion-banner-demo.mp4 --quality high --workers 1 --no-browser-gpu --low-memory-mode --experimental-fast-capture=false
```

В реальном проекте первый шаг — получить URL сайта, размеры/форматы размещения, четыре оффера и ограничения по весу файла. Затем согласовать один статичный hero-frame и только после этого анимировать серию.
